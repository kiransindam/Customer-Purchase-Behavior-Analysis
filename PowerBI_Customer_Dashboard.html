Complete DAX Measure Library — Paste into Power BI Desktop (Modeling → New Measure)
Core Revenue Measures
// Total Revenue Total Revenue = SUM( CustomerTrends[purchase_amount] ) // Average Purchase Amount Avg Purchase = AVERAGE( CustomerTrends[purchase_amount] ) // Total Transactions Total Transactions = COUNTROWS( CustomerTrends ) // Revenue per Subscriber Revenue per Subscriber = DIVIDE( CALCULATE( [Total Revenue], CustomerTrends[subscription_status] = "Yes" ), CALCULATE( [Total Transactions], CustomerTrends[subscription_status] = "Yes" ) )
Customer KPIs
// Unique Customers Customer Count = DISTINCTCOUNT( CustomerTrends[customer_id] ) // % Subscribed % Subscribed = DIVIDE( CALCULATE( [Customer Count], CustomerTrends[subscription_status] = "Yes" ), [Customer Count] ) // % With Discount % Discount Used = DIVIDE( CALCULATE( [Total Transactions], CustomerTrends[discount_applied] = "Yes" ), [Total Transactions] ) // Average Review Rating Avg Rating = AVERAGE( CustomerTrends[review_rating] )
Segment Comparisons
// Revenue for Male customers Revenue Male = CALCULATE( [Total Revenue], CustomerTrends[gender] = "Male" ) // Revenue for Female customers Revenue Female = CALCULATE( [Total Revenue], CustomerTrends[gender] = "Female" ) // Top season revenue Top Season Revenue = MAXX( VALUES( CustomerTrends[season] ), [Total Revenue] ) // Revenue rank by category Category Revenue Rank = RANKX( ALL( CustomerTrends[category] ), [Total Revenue] )
Time & Frequency Intelligence
// Avg days between purchases Avg Freq Days = AVERAGE( CustomerTrends[purchase_frequency_days] ) // High-frequency customers (weekly) Weekly Buyers = CALCULATE( [Customer Count], CustomerTrends[frequency_of_purchases] = "Weekly" ) // Revenue from loyal customers (>25 purchases) Loyal Customer Revenue = CALCULATE( [Total Revenue], CustomerTrends[previous_purchases] > 25 ) // Purchase velocity index Purchase Velocity = DIVIDE( [Total Revenue], AVERAGE( CustomerTrends[purchase_frequency_days] ) )
Advanced Calculations
// Revenue % of total (for % bar charts) Revenue % of Total = DIVIDE( [Total Revenue], CALCULATE( [Total Revenue], ALL( CustomerTrends ) ) ) // Running total revenue Running Revenue = CALCULATE( [Total Revenue], FILTER( ALL( CustomerTrends[previous_purchases] ), CustomerTrends[previous_purchases] <= MAX( CustomerTrends[previous_purchases] ) ) ) // Subscription lift (vs non-subscribed avg) Subscription Lift = DIVIDE( CALCULATE( [Avg Purchase], CustomerTrends[subscription_status] = "Yes" ), CALCULATE( [Avg Purchase], CustomerTrends[subscription_status] = "No" ) ) - 1
KPI Card Measures
// Dynamic title for KPI card Revenue Card Title = "Total Revenue: $" & FORMAT( [Total Revenue], "#,##0" ) // Color indicator (for conditional formatting) Rating Color = IF( [Avg Rating] >= 4.0, "#10b981", IF( [Avg Rating] >= 3.5, "#f59e0b", "#f43f5e" ) ) // Formatted avg purchase for card Avg Purchase Label = "$" & FORMAT( [Avg Purchase], "0.00" ) // Transaction growth placeholder Txn Formatted = FORMAT( [Total Transactions], "#,##0" ) & " orders"
Step-by-step Power BI Desktop setup — Follow in order
01
Download & Install Power BI Desktop
Go to microsoft.com/en-us/power-bi and click "Download free". Install the .msi file. Sign in with a Microsoft account (free). Power BI Desktop is 100% free to use.
Free Tool
02
Import the CSV Dataset
Click Home → Get Data → Text/CSV. Navigate to customer_trends_analysis.csv. In the preview dialog, click "Transform Data" to open the Power Query Editor before loading.
Data Source
03
Clean Data in Power Query Editor
Select age_group column → Transform → Replace Values: change "Aduit" to "Adult". Set purchase_amount as Whole Number. Set review_rating as Decimal Number. Click Close & Apply.
Power Query
04
Rename the Table
In the Fields pane on the right, right-click the table name → Rename. Type "CustomerTrends". This matches all the DAX measures in this guide and avoids formula errors.
Data Model
05
Create DAX Measures
Click Modeling tab → New Measure. Copy-paste each DAX formula from the "DAX Measures" tab of this dashboard. Start with Total Revenue, Avg Purchase, and Customer Count.
DAX
06
Build KPI Cards (Page 1)
Select the Card visual from Visualizations panel. Drag [Total Revenue] into the Fields well. Repeat for Avg Purchase, Avg Rating, Customer Count, % Subscribed. Arrange 5 cards in a top row.
Visuals
07
Add Charts — Category & Season
Clustered Column Chart: X-axis = category, Y-axis = [Total Revenue]. Add a second chart: X-axis = season, Y-axis = [Total Revenue]. For colors, use the Format pane → Data colors.
Bar Charts
08
Add Donut Charts
Donut Chart visual: Legend = payment_method, Values = [Total Revenue]. Repeat for gender, subscription_status, and discount_applied. Enable data labels in the Format pane.
Donut
09
Build the Matrix Visual
Select Matrix visual. Rows = category. Columns = season. Values = [Total Revenue]. In Format → Cell elements → enable Background color → choose a gradient (white to blue). This creates a heatmap effect.
Matrix
10
Add Slicers for Interactivity
Slicer visual: Field = gender. Add more slicers for season, age_group, subscription_status, discount_applied. Change slicer style to "Dropdown" in the Format pane for a compact layout.
Slicers
11
Theme & Branding
Click View → Themes → browse built-in themes or import a custom JSON theme. Add a title text box at the top. Set canvas background: Format → Wallpaper → choose a subtle dark or light color.
Design
12
Publish to Power BI Service
Click File → Save As → save the .pbix file. Then click Home → Publish → select "My Workspace". Open app.powerbi.com to view online. Use Share to send to others via email or link.
Publish
